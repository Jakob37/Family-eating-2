# Agent Workflow

- After completing any requested, self-contained change in this repository, stage the relevant files and create a git commit before responding.
- After creating that commit, push the current branch to the configured remote before responding.
- Make one commit per completed change unless the user explicitly asks for a different grouping.
- Do not amend existing commits unless the user explicitly requests it.
- Every committed change should include a version bump in `pubspec.yaml`.
- Keep any in-app version file in sync with `pubspec.yaml`.
- Keep `CHANGELOG.md` updated in the same change.
